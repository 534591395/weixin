  1. 微信小程序介绍
    官方解说：
        微信小程序是一种全新的连接用户与服务的方式，它可以在微信内被便捷地获取和传播，同时具有出色的使用体验。
    详细解说地址： https://mp.weixin.qq.com/debug/wxadoc/introduction/index.html?t=20161107
  1. 微信小程序的限制 
    1). 不支持HTML、没有 Dom。网页用的 JS、CSS 基本要全部重写，WXML 的语法和 HTML 差异还挺大，基本是一个个照着手册的属性去改。CSS 选择器不支持级联。    
2). 小程序源码打包后的大小限制为1M，超大传不上去。单次通过 wx.request传输的数据最大也是1M。    
3). MINA 框架实现的 tab bar，最多5个 tab；通过 wx. navigateTo 推入后台的页面最多5层，超过会无法打开新页面。    
4). 不支持 A 标签，无法打开普通网页。    
5). 不能分享到朋友圈，只能用户扫描或者用户搜索，推广成本上升。
    ● 整体上来讲，小程序本身被设计为处理简单逻辑的「工具型」应用；同时具有很强的内容封闭性。

  1. 微信小程序开发 微信小程序属于 js-native 范畴，类似 react-native。
    1）. 下载微信提供的开发者工具后，按照提示，本地创建项目。默认已创建了一些必须的文件： app.js 、app.json、app.wxss。其中，.js后缀的是脚本文件，.json后缀的文件是配置文件，.wxss后缀的是样式表文件。 说明：
    A： app.js
    app.js是小程序的脚本代码。我们可以在这个文件中监听并处理小程序的生命周期函数、声明全局变量。调用框架提供的丰富的 API，如本例的同步存储及同步读取本地数据。

    B: app.json
    app.json 是对整个小程序的全局配置。我们可以在这个文件中配置小程序是由哪些页面组成，配置小程序的窗口背景色，配置导航条样式，配置默认标题。

         {
       "pages":[
         "pages/index/index",
         "pages/logs/logs"
       ],
       "window":{
         "backgroundTextStyle":"light",
         "navigationBarBackgroundColor": "#fff",
         "navigationBarTitleText": "WeChat",
         "navigationBarTextStyle":"black"
       }
     }

    微信小程序中的每一个页面的【路径+页面名】都需要写在 app.json 的 pages 中，且 pages 中的第一个页面是小程序的首页。

    C: app.wxss
    app.wxss是整个小程序的公共样式表，可以在页面组件的 class 属性上直接使用 app.wxss 中声明的样式规则。 

         /**app.wxss**/
     .container {
       height: 100%;
       display: flex;
       flex-direction: column;
       align-items: center;
       justify-content: space-between;
       padding: 200rpx 0;
       box-sizing: border-box;
     }

    2）. 创建页面
        每一个小程序页面是由同路径下同名的四个不同后缀文件的组成，如：index.js、index.wxml、index.wxss、index.json。.js后缀的文件是脚本文件，.json后缀的文件是配置文件，.wxss后缀的是样式表文件，.wxml后缀的文件是页面结构文件。 index.wxml：页面的结构文件

       <!--index.wxml-->
   <view class="container">
     <view  bindtap="bindViewTap" class="userinfo">
       <image class="userinfo-avatar" src="" background-size="cover"></image>
       <text class="userinfo-nickname"></text>
     </view>
     <view class="usermotto">
       <text class="user-motto"></text>
     </view>
   </view>

    index.js：页面的脚本文件
    index.json：页面的配置文件
    页面的配置文件是非必要的。当有页面的配置文件时，配置项在该页面会覆盖 app.json 的 window 中相同的配置项。如果没有指定的页面配置文件，则在该页面直接使用 app.json 中的默认配置。

  1. 微信小程序前端数据采集初探
    获取用户操作，一般前端加上监听事件。
    微信小程序中事件分为：冒泡事件和非冒泡事件。
    绑定事件时，bind事件绑定不会阻止冒泡事件向上冒泡，catch事件绑定可以阻止冒泡事件向上冒泡。

    全局监听:
        在页面最外层的父view标签上，绑定事件 bindtap="mixpanelLocal"， bind开头的事件会向上冒泡，这样就能监听所有用户点击事件, 如点击 ”堂食“菜单，mixpanelLocal方法中就能获取事件对象。
    但在 ”堂食“菜单阻止了事件冒泡，比如：绑定了事件 catchtap="catch",则全局监听就无法获取该动作。
    例子是一个在线外卖订单系统。
    首页中添加了一个全局监听： bindtap

       <view class="container flex-wrap flex-direction-col" bindtap="mixpanelLocal" id="page-index" data-type="页面">
         //菜单栏
         <navigator url="../components//" hover-class="navigator-hover" id="" data-type="菜单"></navigator>
   </view>

    点击菜单 bindtap 获取到的事件对象属性列表：

    1. type: tap     // 事件类型
2. timeStamp: 38865  // 事件生成时的时间戳
3. target: {"id":"堂食","offsetLeft":0,"offsetTop":165,"dataset":{"type":"菜单"}} // 触发事件的组件的一些属性值集合， 这里是点击的那个菜单 “食堂”
4. currentTarget: {"id":"page-index","offsetLeft":0,"offsetTop":0,"dataset":{"type":"页面"}} // 当前组件的一些属性值集合,  这里是 页面父容器
5. detail: {"x":59,"y":240} //其它数据
6. touches: [{"identifier":0,"pageX":59,"pageY":241,"clientX":59,"clientY":241}]
7. changedTouches:  [{"identifier":0,"pageX":59,"pageY":241,"clientX":59,"clientY":241}]

    采集:

    当前页引入采集库    
var mixpanel = require('../../utils/mixpanel.local.js').mixpanelLocal;    
//方法mixpanelLocal中改造    
mixpanelLocal： function(e) {
    mixpanel.init(e)
} 
每次点击后，即发送请求。
